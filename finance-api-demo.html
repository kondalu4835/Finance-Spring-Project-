Finance Dashboard API v1.0.0
A complete REST API backend for a finance dashboard system with JWT authentication, role-based access control, financial records management, and analytics endpoints.

⚡ How to Use This API
1
Login — POST to /api/auth/login with email + password. Get back a token string.
2
Authorize — Add header Authorization: Bearer <token> to all protected requests. In live Swagger: click the 🔒 Authorize button and paste the token.
3
Explore endpoints — GET /api/dashboard/summary to see totals. GET /api/records to browse financial data with filters. Admin credentials unlock record/user management.
4
Live Swagger UI → https://finance-spring-project.onrender.com/swagger-ui.html — try every endpoint interactively in the browser with no setup required.

Authentication

3 endpoints
POST /api/auth/login PUBLIC Login and receive JWT token ▾

Submit email and password. Returns a signed JWT token valid for 8 hours. Use the token in the Authorization: Bearer <token> header for all protected endpoints.

Request Body
JSON 
{
  "email": "admin@finance.dev",
  "password": "Admin@1234"
}
Response 200
JSON 
{
  "success": true,
  "data": {
    "token": "eyJhbGciOi...",
    "type": "Bearer",
    "user": {
      "id": 1,
      "name": "Alice Admin",
      "email": "admin@finance.dev",
      "role": "ADMIN",
      "active": true
    }
  }
}
Responses
200Login successful — returns JWT token and user profile
401Invalid credentials
400Validation error (missing/invalid fields)
POST /api/auth/register PUBLIC Register — always gets VIEWER role ▾

Creates a new user account. Self-registered users always receive the VIEWER role. An Admin must use POST /api/users to create accounts with elevated roles.

Request Body
JSON
{
  "name": "John Doe",
  "email": "john@example.com",
  "password": "Password@123"
}
Validation Rules
FieldRule
nameRequired, not blank
emailValid email format, unique
passwordMin 8 characters
GET /api/auth/me ANY AUTH Get current user profile ▾

Returns the profile of the currently authenticated user based on the JWT token in the Authorization header.

Response 200
JSON
{
  "success": true,
  "data": {
    "id": 1,
    "name": "Alice Admin",
    "email": "admin@finance.dev",
    "role": "ADMIN",
    "active": true,
    "createdAt": "2026-04-06T10:30:46"
  }
}

Financial Records

5 endpoints
GET /api/records ANY AUTH List records with filters & pagination ▾

Returns a paginated list of financial records. All filters are optional and can be combined.

Query Parameters
ParamTypeRequired
typeINCOME | EXPENSEoptional
categorystringoptional
dateFromYYYY-MM-DDoptional
dateToYYYY-MM-DDoptional
pageintegerdefault: 0
sizeintegerdefault: 20
Response 200
JSON
{
  "success": true,
  "data": {
    "content": [{
      "id": 1,
      "amount": 5000.00,
      "type": "INCOME",
      "category": "Salary",
      "date": "2024-04-01",
      "notes": "April salary"
    }],
    "page": 0,
    "size": 20,
    "totalElements": 42,
    "totalPages": 3
  }
}
GET /api/records/{id} ANY AUTH Get single record by ID ▾
Path Parameters
ParamTypeRequired
idLongrequired
Responses
200Record found — returns full record object
404Record not found or soft-deleted
POST /api/records ADMIN Create a new financial record ▾
Request Body
JSON
{
  "amount": 5000.00,
  "type": "INCOME",
  "category": "Salary",
  "date": "2024-06-01",
  "notes": "June salary"
}
Validation Rules
FieldRule
amountRequired, > 0.01
typeRequired: INCOME or EXPENSE
categoryRequired, not blank
dateRequired, ISO date
notesOptional, max 1000 chars
Responses
201Record created successfully
403Forbidden — requires ADMIN role
400Validation failed
PATCH /api/records/{id} ADMIN Partial update of a record ▾

Only fields provided in the request body are updated. All fields are optional.

Request Body (all fields optional)
JSON
{
  "amount": 6000.00,
  // only update amount, leave other fields unchanged
}
DEL /api/records/{id} ADMIN Soft-delete a record ▾

Records are never permanently removed. A deletedAt timestamp is set instead. All list/get queries filter on deletedAt IS NULL — standard practice for financial data to maintain an audit trail.

Responses
200Record soft-deleted successfully
404Record not found
403Forbidden — requires ADMIN role
